# Music Genre Classification — Design Spec

**Date:** 2026-05-15
**Status:** Approved
**Dataset:** CCMUSIC Music Genre Dataset (ccmusic-database/music_genre)

---

## 1. Problem Statement

Train a deep learning model to classify music into 16 fine-grained genres from the CCMUSIC dataset. The model must support local inference on raw `.mp3` files as well as serve as a platform for research experimentation.

---

## 2. Dataset

| Property | Value |
|----------|-------|
| Source | HuggingFace: `ccmusic-database/music_genre` |
| Modality | Mel spectrogram images (11.4s segments) from ~1,700 tracks |
| Samples | Train 29,100 / Val 3,637 / Test 3,638 |
| Labels | 3-level hierarchy; target = Level 3 (16 fine-grained classes) |
| License | MIT |
| Original audio | 22 kHz sample rate, 270–300s per piece (not included — spectrograms only) |

### 16 Fine-Grained Labels

Classic family: Symphony, Opera, Solo, Chamber
Pop family: Pop_vocal_ballad, Adult_contemporary, Teen_pop
Dance family: Contemporary_dance_pop, Dance_pop
Indie family: Classic_indie_pop, Chamber_cabaret_and_art_pop
Soul/R&B: Soul_or_r_and_b
Rock family: Adult_alternative_rock, Uplifting_anthemic_rock, Soft_rock, Acoustic_pop

## 3. Architecture

### 3.1 Primary Model: AST (Audio Spectrogram Transformer)

ViT-Base backbone adapted for spectrograms, with AudioSet pretrained weights.

| Component | Configuration |
|-----------|---------------|
| Input | Mel spectrogram, 1 channel, 128 mel bins × time frames (resize to 224×224) |
| Patch embedding | 16×16 patches, flattened → 768-dim |
| Position encoding | Learnable positional embeddings |
| Transformer | 12 layers, 12 attention heads, 768 hidden dim, 3072 MLP dim |
| Classification head | Linear(768 → 16), softmax |
| Pretraining | AudioSet (527 classes, ~2M clips), weights from HuggingFace `MIT/ast-finetuned-audioset` |

### 3.2 Baseline: ResNet50

Standard ResNet50 with ImageNet pretrained weights. Input: 3-channel mel spectrogram (replicated). Classification head: Linear(2048 → 16).

Used only for comparison; not required for final model.

## 4. Training Strategy

### 4.1 AST Fine-Tuning

| Parameter | Value | Notes |
|-----------|-------|-------|
| Optimizer | AdamW | Standard Transformer optimizer |
| Learning rate (backbone) | 5e-5 | Low LR — pretrained weights, careful tuning |
| Learning rate (head) | 5e-4 | Higher LR — randomly initialized |
| LR schedule | Cosine annealing + 5 epoch linear warmup | warmup = `warmup_steps / total_steps` percent |
| Batch size | 32 | Fits 12GB+ VRAM |
| Epochs | 30 max, early stop patience=5 | Monitor val loss |
| Weight decay | 0.01 | |
| Loss | CrossEntropyLoss with label smoothing (0.1) | Smooths boundaries between similar sub-genres |
| Data augmentations | SpecAugment (time mask 10%, freq mask 2 bands), Mixup (α=0.5) | Applied per batch at training time only |

### 4.2 ResNet50 Baseline

| Parameter | Value |
|-----------|-------|
| Optimizer | Adam, lr=1e-4 |
| LR schedule | ReduceLROnPlateau (patience=3, factor=0.5) |
| Batch size | 64 (CNN uses less VRAM) |
| Epochs | 30, early stop patience=5 |
| Augmentations | Same as AST minus Mixup (standard for CNN) |

## 5. Evaluation

### 5.1 Primary Metrics

- **Macro F1** — primary metric (class-balanced, robust to imbalance)
- Accuracy — intuitive reference
- Weighted F1 — sample-weighted, secondary

### 5.2 Analysis Tools

- Confusion matrix (16×16) — identify confusion pairs
- Per-class precision/recall/F1 — surface weak classes
- **Mid-level roll-up accuracy** — map 16 fine-grained predictions back to 9 mid-grained classes and recompute accuracy. Reveals at what hierarchy level the model starts to fail.

### 5.3 Comparison

- AST vs ResNet50 side-by-side on all metrics
- Report in a single table + two confusion matrices

## 6. Inference Pipeline

```
user's mp3 file
    │
    ▼
librosa.load(sr=22050)          # Load audio, resample to 22kHz
    │
    ▼
librosa.feature.melspectrogram( # Extract mel spectrogram
    n_mels=128, hop_length=512,
    n_fft=2048, fmax=11025
)
    │
    ▼
PowerToDB + min-max normalize    # Convert to dB, then scale to [0,1] via (x - min) / (max - min)
    │
    ▼
Resize to 224×224 (bilinear)     # Match training input size
    │
    ▼
Model.forward()                 # AST inference
    │
    ▼
Softmax → {genre: confidence}   # Return top-3 predictions with confidence scores
```

For audio longer than 11.4s: slide an 11.4s window with 50% overlap, average the logits across all windows before softmax. For audio shorter than 11.4s: pad with silence (zero-pad mel bands) to reach 11.4s equivalent time frames.

**Known risk — train/inference mel mismatch:** Training uses the dataset's pre-generated spectrogram JPEGs (generated by the dataset authors with unknown exact extraction params). Inference uses librosa with the params above. These may differ subtly in scaling, colormap, or frequency resolution. Mitigation: (1) The model should learn robust features from the dataset's variety; (2) If inference accuracy is poor, a second-stage fine-tuning on librosa-generated spectrograms from a small held-out set of mp3s can bridge the gap; (3) Normalize both to [0,1] range to reduce scaling differences.

## 10. Out of Scope (YAGNI)

- No multi-spectrogram fusion (CQT, chroma) — mel only in v1
- No ensemble across ResNet + AST
- No real-time audio streaming support
- No web UI / API deployment
- No audio generation or source separation
